🐛 v2.0.1 - Bug Fixes for Calendar & Team Flags Views

## Summary
Fixed critical UI/UX issues in the Calendar and Team Flags views introduced in v2.0.0, improving readability and functionality across all prediction views.

## Bug Fixes

### 1. Calendar View Readability
- **Issue**: Calendar dates were unreadable (white text on light background)
- **Fix**: Increased background opacity from 0.05 to 0.15 for better contrast
- **Fix**: Enhanced day number styling with larger font (18px), bold weight (700), and text shadows
- **Fix**: Improved text shadows for better visibility on gradient backgrounds
- **Impact**: Calendar dates are now clearly visible and easy to read

### 2. Team Flags View Readability  
- **Issue**: Team names were unreadable (white text on light background)
- **Fix**: Added dark semi-transparent backgrounds (rgba(0,0,0,0.6)) to team cards
- **Fix**: Enhanced text shadows for team names and match counts
- **Fix**: Improved contrast for better readability
- **Impact**: Team names and match counts are now clearly visible

### 3. Prediction Button Functionality
- **Issue**: Prediction buttons not working in Calendar and Team Flags views
- **Fix**: Created `openPredictionModalById()` helper function to properly handle match ID parameter
- **Fix**: Updated `createMatchCard()` to call the new helper function
- **Impact**: Prediction buttons now work correctly in all views

### 4. Match Card Consistency
- **Issue**: Match cards in Calendar/Team views didn't match List view styling
- **Fix**: Ensured all views use the same `createMatchCard()` function
- **Fix**: Maintained consistent styling and functionality across all views
- **Impact**: Uniform user experience across all prediction views
